Aide a l'utilisation

{% if user.FAisFA %}

En tant que FA de l'association, vous pouvez:

  • Visualiser la liste de chats sous votre responsabilité (vous pouvez toujours revenir à cette liste en cliquant sur votre nom en haut). La liste indique:
    • Le numéro de registre (cliquer pour accéder/modifier les informations)
    • Le numéro de puce (ou tatouage)
    • Le nom (cliquer pour faire apparaître/disparaître les commentaires)
    • La description du chat: sexe, âge, couleur (les info manquantes sont en jaune)
    • une indication des visites vétérinaires que le chat a eu (V=primo vaccination, 1=1er rappel, 2=2eme rappel, S=stérilisation/castration, P=identification, T=test FIV/Felv, R=rappel annuel, X=soins)
    • l'indication de si le chat est a l'adoption ou pas
  • Modifier pratiquement toutes les informations relatives à vos chats, particulièrement utile pour l'association est de rajouter le numéro de puce lors de l'identification du chat. Pour ce qui en est des commentaires et des descriptions, les commentaires sont internes à ERA et sont utilisés pour indiquer des situations particulières; comme certaines sont récurrentes, nous avons des habitudes qu'il serait utile de suivre:
    • pour un chat qui arrive déjà stérilisé, indiquer "Déjà castré" ou "Déjà stérilisées"
    • si le test FIV est positif, indiquer "FIV+", si le test est négatif, ne rien indiquer
    • si le test FeLV est positif, indiquer "Positif LEUCOSE", si le test est négatif, ne rien indiquer
    • si lors de l'adoption les adoptants ont déjà un nouveau nom a lui donner, indiquer "Nouveau nom : LE NOM EN MAJUSCULES"
    • indiquer les problèmes chroniques/malformations/necessites particulières, exemples: "Diabétique", "Problèmes rénaux", "Amputé patte arrière gauche". "Allergie: nécessite croquettes spéciales", etc.
  • Rajouter des éléments utiles pour passer les annonces d'adoption, considérez dans ce cas que le texte "Description" et la photo risquent d'être diffusées publiquement. Pensez donc à ne pas y mettre des informations personnelles
  • Indiquer ou planifier les visites vétérinaires. Cette fonctionnalité est accessible de la page du chat en question et permet d'indiquer une visite qui a eu lieu ou encore une visite que vous devez effectuer dans le futur. Toute visite planifiée peut être transformée en effectuée simplement en changeant l'indication et en mettant à jour les informations. Vous pouvez à tout moment effacer une visite planifiée en indiquant "ANNULLEE" et en mettant à jour. Les visites effectuées sont enregistrées définitivement et ne peuvent pas être annulées, pensez donc à bien vérifier les informations avant de valider la mise à jour....
    Les visites sont accompagnées de commentaires libres qui sont utiles pour détailler les soins ou les traitements. Pour les visites planifiées, ils peuvent servir d'aide-mémoire
  • A tout moment vous pouvez accéder au calendrier de visites planifiées avec le menu Famille->Visites vétérinaires. Cette page permet aussi d'autres opérations, notamment:
    • passer de "planifiées" à "effectuées" un ensemble de visites effectuées un même jour, même si elle concernent plusieurs chats différentes
    • planifier/indiquer une même visite pour plusieurs chats, particulièrement utile pour une portée de chatons qui vont aller chez le vétérinaire tous au même temps...
    • demander l'autorisation pour une visite et (une fois celle-ci autorisée par un responsable de l'association), imprimer le bon vétérinaire; il suffit pour cette opération de planifier toutes les visites qu'on doit effectuer et ensuite les sélectionner et demander l'autorisation; comme le programme n'alerte pas directement les responsables, pensez à indiquer à un d'eux par mail ou SMS que vous avez besoin d'une autorisation pour le bon veterinaire.
    Le bon vétérinaire n'apparaît pas correctement mis en page à l'écran, mais normalement si on l'imprime sur une page A4 il devrait avoir le bon format.
  • L'historique de toutes les visites que vous avez effectuées est enregistré dans le registre des soins
  • Vous ne pouvez pas indiquer le transfert vers une autre FA, ni rajouter un nouveau chat, ces opérations doivent être effectuées par un responsable de l'association

Opérations possibles pour un responsable FA

  • En tant que responsable FA, vous avez accès aux informations des FA sous votre responsabilité: la liste de ces FA étant accessible avec le bouton correspondant
  • Lorsque vous opérez sur une autre FA, la première ligne de la page vous indique la FA dont vous voyez les données
    • Vous avez accès à leur chats comme s'il s'agissait de vôtres, vous pouvez donc mettre à jour les informations, planifier/indiquer des visites vétérinaires, etc. Vous ne pouvez pas transférer des chats entre FA, celui-ci doit être effectuée par un responsable de l'association
  • Vous pouvez visualiser le calendrier des visites de cette FA ainsi que son registre des soins
  • Vous avez aussi accès aux les listes des chats actuellement au refuge aussi bien que aux récemment adoptés ou décédés
{% endif %} {% if user.FAisOV %}

Opérations possibles pour un superviseur

  • Visualiser les informations de toutes les FA (liste des chats, visites)
  • Visualiser la liste de tous les chats ou celle limitée aux chats marques "adoptable"
  • Visualiser les listes des chats adoptés ou décédés
  • Vous ne pouvez en aucun cas modifier les informations relatives aux chats, ni les déplacer
{% endif %} {% if user.FAisADM %}

Opérations possibles un gestionnaire registre

  • Rajouter un chat dans le système, soit manuellement soit par copier-coller d'un dossier Refugilys (il y a un format spécifique à respecter!)
  • Effectuer toutes les opérations des autres rôles, en ayant donc accès à toutes le informations de tous les chats en FA, avec la possibilité des les modifier
  • Transférer les chats entre FA, y compris "FA spéciales" (adoptés/décédés/historique), donc par exemple un chat indique décédé par erreur peut être remis dans une FA
  • Lors de l'enregistrement d'une adoption ou décès, vous pouvez transférer le chat dans l'historique
  • Effacer définitivement du système un chat de l'historique: !!! ATTENTION !!! cette opération est irréversible et efface aussi toutes les informations relatives aux visites vétérinaires effectuées sur ce chat, en le faisant donc disparaître de tous les registres de soin des FA concernées!
  • Vous pouvez exporter les dernières modification des informations sur les chats ou exporter l'intégralité de la base de données. Le format utilise est le même que dans le cas d'importation des dossiers Refugilys (et il n'est pas très lisible, vu qu'il est conçu pour l'échange de données et non pas pour la lecture; si vous voulez un tableau lisible, l'export CSV des listes accessible en tant que superviseur est un meilleur choix)
  • Rajouter ou modifier les informations relatives à un utilisateur: seulement certains types d'utilisateurs peuvent être rajoutes (ou modifies) par l'interface, les utilisateurs spéciaux (adoptions, décès, refuge, historique, GR) ne peuvent pas être modifies. Cette même interface permet de rajouter une clinique vétérinaire dans la liste, de façon à qu'elle apparaisse dans les choix possibles pour les visites veterinaires
  • Effectuer des opérations système que vous ne devez pas effectuer; donc faites comme si cet option du menu n'existait pas
